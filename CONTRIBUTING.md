# Contributing

Thanks for taking the time to contribute. Corrections, new resources, structural improvements, and broken link reports are all welcome.

---

## Ways to contribute

- **Fix content:** Typos, outdated information, unclear explanations
- **Add resources:** Tools, labs, articles, videos — relevant and well-maintained ones only
- **Suggest new sections:** Topics that are missing or underdeveloped
- **Report errors:** Broken links, stale information, factual mistakes

---

## Reporting issues

Open a [GitHub Issue](https://github.com/Nervi0zz0/soc-analyst-starter/issues) and check for duplicates first.

For **bugs or errors**, include the section where the problem is, what's wrong, and ideally a suggested fix.
For **resource suggestions**, include the name, URL, and a brief note on why it's useful for a SOC L1 analyst.

Use labels where relevant: `bug`, `enhancement`, `broken-link`, `new-resource`.

---

## Submitting a pull request

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/soc-analyst-starter.git`
3. Create a descriptive branch:
   ```bash
   git checkout -b fix-siem-broken-link
   ```
4. Make your changes
5. Commit with a clear message:
   ```bash
   git commit -am "fix: correct broken link in SIEM section"
   # Other examples:
   # docs: add TryHackMe room to endpoint analysis
   # feat: add DFIR resources section
   ```
6. Push your branch: `git push origin fix-siem-broken-link`
7. Open a pull request against `main`. Include a clear description of what you changed and why. Reference any related issue with `Closes #NUMBER`.

---

## Guidelines

- Quality over quantity — prefer established, actively maintained resources
- Keep descriptions concise and factual
- Follow the formatting used in the existing sections

---

For any questions about the process, open an issue.
